# LoraRouter

LoraRouter was my attempt to build a lightweight routing and communication layer for LoRa and LoRaWAN traffic.

The goal was to work closer to the packet and gateway layer than most application-level tooling, handling message flow, forwarding, and routing behavior directly rather than depending entirely on external network infrastructure. I was interested in how LoRaWAN-style communication could be managed, relayed, and coordinated through a small custom service.

The repository explores packet parsing, gateway-facing communication, routing logic, and low-level handling of LoRa network traffic. It reflects an effort to better understand and control how messages moved through a LoRa-based system, with the emphasis on infrastructure and protocol behavior rather than end-user applications.

## Concepts explored

- LoRa and LoRaWAN packet handling
- routing and forwarding logic
- gateway-facing communication
- lightweight network infrastructure for radio traffic
- protocol-level experimentation around message flow
